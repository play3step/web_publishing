interface란?

interface에는 총 3가지가 있습니다.
Software Interface, Hardware Interface, User Interface

사전적 의미로 다른 사물이나 시스템 간에 커뮤니케이션(소통)이 가능하도록 설계한 상호 작용 방식

Software Interface

Software interface란?

정의 : 소프트웨어 간의 통신을 위해 메시지를 전달하는 방식 등이 결정된 것

특징 : 시스템 자원에 대한 기능성과 안정성 제공

예시 : 운영체제와 하드웨어들 사이의 인터페이스, 운영체제와 응용 프로그램이 상호 작용하는 인터페이스, 객체 지향 프로그래밍에서 응용 내의 객체들이 서로 상호 작용할 수 있도록 하는 메소드

Hardware Interface

Hardware interface란?

정의 : 정보 기기의 하드웨어 간 통신을 위해서 존재하는 입출력 포트의 전기적 연결 장치인 커넥터, 신호의 송수신 방법(프로토콜) 등

특징 : 컴퓨터 관련 버스, 저장장치 및 다양한 입출력 기기들 내에 존재, 병렬연결과 직렬 연결로 나누어짐

예시 : USB, 플러그, 커넥터, 카드

User Interface

User interface란?

정의 : 기계, 특히 컴퓨터와 사용자(인간) 사이의 상호 작용과 정보 교환을 위한 인터페이스

특징 : 크게 입력과 출력 두 가지 방법 제공, 사용자가 필요한 요소를 쉽게 찾고 사용하며 정확한 결과를 손쉽게 얻어낼 수 있도록 설계

예시 : GUI, WUI, TUI …

CSR/SSR 정의와 장단점

렌더링이란??

브라우저 렌더링이란 브라우저가 서버로부터 요청해 받은 내용을 브라우저 화면(View)에 표시해주는 작업을 말합니다. HTML, CSS, JavaScript 문서를 전달받아 브라우저 엔진이 각 문서를 해석해 브라우저 화면을 그려주는 것을 말합니다.

1. CSR란?

CSR(Client Side Rendering)

SSR과 반대로 렌더링이 클라이언트쪽에서 일어나

2. SSR란?

SSR(Server Side Rendering)

서버쪽에서 렌더링 준비를 끝마친 상태로 클라이언트에 전달하는 방식을 말합니다.

장점/단점

CSR SSR
장점 단점 장점 단점
  • 네이티브 앱과 비슷한 빠른 인터렉션을 구현할 수 있다. (새로고침이 발생하지 않아 사용자가 네이티브 앱과 비슷한 경험을 할 수 있다.)
  • 첫 페이지 로딩 속도가 서버 사이드 렌더링에 비해 다소 느립니다.

  • 검색엔진최적화(SEO)에 대한 추가 보완 작업이 필요하다. (포털사이트 검색엔진 크롤러가 웹사이트에 대한 데이터를 제대로 수집하지 못하는 경우가 발생할 수 잇다.)
  • 첫 페이지 로딩 속도가 클라이언트 사이드 렌더링에 비해 더 빠름

  • 검색엔진최적화(SEO)가 가능함
  • 초기 로딩 이후 페이지 이동 시 속도가 다소 느립니다.

    • 사용 권장 예시

      CSR

    • 네트워크가 빠를 때
    • 서버의 성능이 좋지 않을 때
    • 사용자에게 보여줘야 하는 데이터의 양이 많을 때
    • 웹 어플리케이션에 사용자와 상호 작용 할 것들이 많을 때(아예 렌더링이 되지 않아 사용자의 행동을 막는것이 경험에 오히려 유리합니다.)

      • SSR

    • 네트워크가 느릴 때 CSR은 한번에 모든 것을 불러오지만 SSR는 각 페이지마다 나누어 불러오기 때문
    • 최초 로딩이 빨라야 하는 사이트를 개발 할 때
    • 메인 스크립트가 크고 로딩이 엄청 느릴 때(CSR은 메인스크립트가 로딩이 끝나면 API로 데이터 요청을 보낸다. 하지만 SSR은 한번의 요청에 아예 렌더가 가능한 페이지가 돌아온다.)
    • 웹 사이트가 상호작용이 별로 없을 때
    MPA/SPA 정의 및 장단점

    1. MPA란?

    MPA(Multiple Page Application)

    여러 개(Single)의 Page로 구성된 Application이다.

    MPA는 SSR(Server Side Application) 방식으로 렌더링한다. 새로운 페이지를 요청할 때마다 서버에서 렌더링된 정적 리소스(HTML, CSS, JavaScript)가 다운로드된다. 페이지 이동하거나 새로고침하면 전체 페이지를 다시 렌더링한다.

    2. SPA란?

    SPA(Single Page Application)

    한 개(Single)의 Page로 구성된 Application이다.

    SPA는 CSR(Client Side Rendering) 방식으로 렌더링한다. 단 한 번만 리소스(HTML, CSS, JavaScript)를 로딩한다. 그 후에는 데이터를 받아올 때만 서버와 통신한다.

    장점/단점

    MPA SPA
    장점 단점 장점 단점
  • SEO 관점에서 유리하다.(MPA는 완성된 형태의 HTML 파일을 서버로부터 전달받는다. 따라서 검색엔진이 페이지를 크롤링하기에 적합하다.)
  • 첫 로딩 매우 짧다.(서버에서 이미 렌더링해 가져오기 때문이다.)
  • 페이지 이동시 불필요한 템플릿도 중복해서 로딩 (성능)

  • 서버의 템플릿 연산을 클라이언트로 분산 (성능)
  • 필요한 리소스만 부분적으로 로딩 (성능)

  • 검색엔진최적화(SEO)가 가능함
  • JavaScript 파일을 번들링해서 한 번에 받기 때문에 초기 구동 속도가 느리다. (Webpack의 code splitting으로 해결 가능)
  • 보안 이슈 (프론트엔드에 비즈니스 로직 최소화)
